01 · Quick start & theming

<html data-pd-theme="dark">            <!-- DARK IS DEFAULT -->
  <head>
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <script src="theme.js"></script>        <!-- optional persistence -->
    <link rel="stylesheet" href="styles.css">  <!-- the ONLY stylesheet -->
  </head>
The whole system flips on one attribute — data-pd-theme="dark" (default) or "light". Use semantic variables (var(--text), var(--surface), var(--accent)) in product code; never hard-code hex. No build step, no npm, no CSS framework.

12 · Responsive & dependencies

Responsive by default — always include the viewport meta. Breakpoints: 900px tablet, 600px mobile, 380px small.

SurfaceBehavior
App shellFull sidebar ≥900 → icon rail 600–900 → top nav <600
Stat gridsAuto-fit → 2-up → 1-up
TablesScroll horizontally inside their card <600
Cards / orbs / chartsStack & scale down; tap targets ≥44px
